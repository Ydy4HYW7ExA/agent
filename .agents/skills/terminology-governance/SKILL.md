# Terminology Governance

对象一旦开始稳定，名字就开始变得危险。临时叫法在交流早期很有用；一旦进入长期文档，它会把对象边界一起带散。同一个对象在不同文件里有不同主词，读者很快就分不清这是故意区分，还是作者没有把名字收住。这个 skill 要收住的，正是对象一致性。

这里需要回答四个问题：哪些词值得成为正式术语，哪些词只停留在局部，哪些词退为别名，哪些词进入共享总表。共享总表保持收紧状态。它只收真正跨多个 skill 的对象，或者已经成为系统级入口对象的词。

## 使用约束

- 一个正式术语命名一个稳定对象
- 共享术语总表只收跨多个 skill 或系统级入口对象
- 术语一旦替换，关键入口尽快形成单一主表述
