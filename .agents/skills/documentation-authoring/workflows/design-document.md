# Design Document

## Purpose

设计一篇真正能承载对象的文档。

## Use When

- 需要新写一篇长期文档
- 现有文档内容很重要，结构仍然松散
- 需要决定正文和结构化元素如何分工

## Inputs

- 文档对象
- 目标读者
- 需要承载的规则、推导或结构

## Steps

1. 先写场景引入，让读者知道问题从哪里逼出来。
2. 所有术语在第一次出现时先给定义，再继续使用。
3. 用自然段写清对象、边界、推导和后果。
4. 再补结构化元素，把目录、状态、依赖、输入输出或落点压实。
5. 删掉只起分栏作用、却没有独立信息量的小节。
6. 清掉黑话、作者手势和防御式转折句。
7. 回看整篇文档，确认读者先看到的是对象。

## Outputs

- 一篇结构与正文配合稳定的文档

## Verification

- 场景是否真的逼出正文对象
- 术语首现是否都已经给出定义
- 结构化元素是否承担了它该承担的硬边界
- 是否仍然存在大量只起索引作用的空段落
- 是否还残留黑话或防御式转折句
