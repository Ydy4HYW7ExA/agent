# Documentation Authoring

文档会滑向两种坏形态。第一种坏形态把内容切成很多小片，每片都很规整，读者却要自己把对象重新拼起来。第二种坏形态只有长段说明，没有结构化元素，导致边界、输入输出、目录结构和关系约束全靠读者在段落里捞。稳定文档要避开这两种极端。

这里要收住六件事：信息落点、术语首现、正文展开、结构化元素、草稿工件、草稿退出。其他四个 skill 最终都会落回文档，所以这六件事会不断反复出现。

## 使用约束

- 反对的是信息碎裂，不反对按对象分层
- 术语第一次出现时先给定义，再继续使用
- 自然段负责完整判断，结构化元素负责压实边界
- 常用词优先；黑话、作者自造词和带姿态的包装词不进入正文
- 防御式转折句不进入正文，例如连续使用“不是……而是……”
- 草稿工件可以存在；它们不和正式文档争夺真值
